<h1>Technical Writing</h1>
<h2>Overview</h2>
<p>Technical writing is the art of communicating complex information clearly and accurately to a specific audience. Whether writing API documentation, tutorials, or internal guides, this skill is essential for any knowledge worker who needs to share expertise with others.</p>
<h2>Proficiency Levels</h2>
<h3>Novice</h3>
<ul>
<li>Can write basic step-by-step instructions</li>
<li>Understands the importance of audience awareness</li>
<li>Uses simple, clear language</li>
</ul>
<h3>Competent</h3>
<ul>
<li>Structures documents logically</li>
<li>Creates effective headings, lists, and formatting</li>
<li>Anticipates reader questions</li>
<li>Uses diagrams and visuals when helpful</li>
</ul>
<h3>Proficient</h3>
<ul>
<li>Adapts tone and depth for different audiences</li>
<li>Creates comprehensive documentation systems</li>
<li>Teaches others to write better documentation</li>
<li>Edits and improves others' technical writing</li>
</ul>
<h3>Expert</h3>
<ul>
<li>Develops documentation standards for organizations</li>
<li>Creates style guides and templates</li>
<li>Innovates in documentation formats and tools</li>
<li>Recognized as a go-to resource for documentation</li>
</ul>

<h2>Learning Path</h2>
<h3>Phase 1: Foundations</h3>
<ul>
<li>[x] Read &quot;The Elements of Style&quot; by Strunk &amp; White</li>
<li>[x] Study good documentation examples (Stripe, Twilio)</li>
<li>[x] Practice the <a href="/garden/dev_admin/concept/the-feynman-technique/" class="internal-link">The Feynman Technique</a> for explanations</li>
</ul>
<h3>Phase 2: Practice</h3>
<ul>
<li>[ ] Write 10 tutorials or blog posts</li>
<li>[ ] Create comprehensive docs for personal project</li>
<li>[ ] Get feedback from target audience</li>
</ul>
<h3>Phase 3: Mastery</h3>
<ul>
<li>[ ] Develop a personal style guide</li>
<li>[ ] Mentor others on documentation</li>
<li>[ ] Create video tutorials combining writing and presentation</li>
</ul>
