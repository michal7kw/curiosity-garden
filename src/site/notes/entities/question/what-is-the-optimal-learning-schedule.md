<h1>What Is The Optimal Learning Schedule?</h1>
<h2>The Question</h2>
<p>How should I structure my learning time for maximum skill acquisition? What's the optimal balance of focused learning, practice, review, and rest?</p>
<h2>Context</h2>
<p>With limited time and energy, scheduling is crucial. This question combines insights from <a href="/garden/dev_admin/course/learning-how-to-learn/" class="internal-link">Learning How to Learn</a>, <a href="/garden/dev_admin/resource/deep-work/" class="internal-link">Deep Work</a>, and <a href="/garden/dev_admin/concept/spaced-repetition/" class="internal-link">Spaced Repetition</a> into a practical daily system. It's particularly relevant for complex skills like <a href="/garden/dev_admin/skill/python-programming/" class="internal-link">Python Programming</a> that require both conceptual understanding and practical application.</p>
<h2>Current Understanding</h2>
<p>Several principles seem to guide optimal scheduling:</p>
<ol>
<li><strong>Spaced practice beats massed practice</strong>: Shorter sessions spread over time outperform long cram sessions</li>
<li><strong>Sleep is non-negotiable</strong>: <a href="/garden/dev_admin/question/how-does-sleep-affect-learning/" class="internal-link">How Does Sleep Affect Learning</a> shows sleep is essential for consolidation</li>
<li><strong>Interleaving helps</strong>: Mixing different topics/skills in a session improves long-term retention</li>
<li><strong>Deep work has limits</strong>: Most people can sustain 4-6 hours of deep cognitive work per day</li>
<li><strong>Rest is productive</strong>: Diffuse mode thinking during rest helps solve problems</li>
</ol>
<h2>Hypotheses</h2>
<h3>Hypothesis 1: Morning is best for new/hard material</h3>
<ul>
<li><strong>Description</strong>: Cognitive resources are highest in the morning, making it ideal for challenging new content</li>
<li><strong>Evidence for</strong>: Research on circadian rhythms; <a href="/garden/dev_admin/resource/deep-work/" class="internal-link">Deep Work</a> recommends morning focus</li>
<li><strong>Evidence against</strong>: Individual chronotypes vary; night owls may differ</li>
</ul>
<h3>Hypothesis 2: Optimal session length is 25-90 minutes</h3>
<ul>
<li><strong>Description</strong>: Shorter than 25 min doesn't allow deep engagement; longer than 90 min leads to diminishing returns</li>
<li><strong>Evidence for</strong>: Pomodoro technique; ultradian rhythms research</li>
<li><strong>Evidence against</strong>: Some tasks benefit from longer uninterrupted blocks</li>
</ul>
<h3>Hypothesis 3: Review should happen at consistent daily times</h3>
<ul>
<li><strong>Description</strong>: <a href="/garden/dev_admin/concept/spaced-repetition/" class="internal-link">Spaced Repetition</a> works best as a regular habit, not sporadic practice</li>
<li><strong>Evidence for</strong>: <a href="/garden/dev_admin/resource/atomic-habits/" class="internal-link">Atomic Habits</a> on habit formation; <a href="/garden/dev_admin/resource/anki/" class="internal-link">Anki</a> best practices</li>
<li><strong>Evidence against</strong>: Flexibility might be needed for varied schedules</li>
</ul>

<h2>Current Best Answer</h2>
<p>Based on exploration, here's my current optimal learning schedule framework:</p>
<h3>Daily Structure</h3>
<ol>
<li>
<p><strong>Morning Deep Work Block (2-3 hours)</strong></p>
<ul>
<li>New material, challenging concepts</li>
<li><a href="/garden/dev_admin/concept/the-feynman-technique/" class="internal-link">The Feynman Technique</a> for understanding</li>
<li>No interruptions, phone away</li>
</ul>
</li>
<li>
<p><strong>Midday <a href="/garden/dev_admin/resource/anki/" class="internal-link">Anki</a> Review (20-30 min)</strong></p>
<ul>
<li><a href="/garden/dev_admin/concept/spaced-repetition/" class="internal-link">Spaced Repetition</a> review session</li>
<li>Maintains previous learning</li>
</ul>
</li>
<li>
<p><strong>Afternoon Practice Session (1-2 hours)</strong></p>
<ul>
<li>Apply learning through projects</li>
<li><a href="/garden/dev_admin/skill/python-programming/" class="internal-link">Python Programming</a> coding practice</li>
<li>Interleave different skills</li>
</ul>
</li>
<li>
<p><strong>Evening Light Review (15-30 min)</strong></p>
<ul>
<li>Preview tomorrow's material</li>
<li>Reflect on day's learning</li>
<li>Material consolidates during <a href="/garden/dev_admin/question/how-does-sleep-affect-learning/" class="internal-link">sleep</a></li>
</ul>
</li>
</ol>
<h3>Weekly Structure</h3>
<ul>
<li><strong>Focused days</strong> (4-5): Primary skill development</li>
<li><strong>Interleaving days</strong> (1-2): Review multiple topics</li>
<li><strong>Rest day</strong> (1): Full cognitive rest, allow diffuse processing</li>
</ul>
<h3>Key Principles</h3>
<ul>
<li>Never skip <a href="/garden/dev_admin/resource/anki/" class="internal-link">Anki</a> reviews</li>
<li>Protect morning deep work time</li>
<li>Sleep 7-9 hours consistently</li>
<li>Track and adjust based on results</li>
</ul>
